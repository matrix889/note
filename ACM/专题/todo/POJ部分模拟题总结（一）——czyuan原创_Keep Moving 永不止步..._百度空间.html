POJ部分模拟题总结（一）——czyuan原创

最近都在练模拟题，特别是一些比较复杂或者奇怪的。

列举最近做的些模拟题：POJ 1835 2623 2271 1573 2996 2993 1068 1350 1786 1489 1676 2083 2015 2317 1082 3699 1472

总结下解模拟题的一些技巧：

1.     英语。一般模拟题的题目都很长，叙述都很繁琐，目的就是要吓到你。这时候英语好就是很大优势了，读懂题意基本就最对一半了，所以还是要好好学英语的～～

2.     信心。一般模拟题都是以超长的题目和不小的代码量来吓人，如果你想解决这题，就一定要有必AC的信心，相信自己只要写就一定会过。模拟题是出给胆大心细的人做的～～

3.     耐心。模拟题的代码量一般都很大，所以写时一定要有耐心，特别是最后输出格式的处理要认真（模拟题挺爱的输出上做文章的...）。

-------------------------------------------------------------------------------------------------------------

推荐几道题目吧：

POJ 1835 宇航员

一道不错的模拟题，对于空间感要求比较高，我们可以先把所有可能旋转的情况列举出来。

用left[i][j]表示head(头顶方向)为i，d(面朝方向)为j，向左旋转后的方向.

同样定义right, up, down, head_up, head_down数组.

然后按要求模拟即可.

POJ 1489 Egyptian Multiplication

就是模拟题，但有点要注意，即计算过程中数值有可能超过99999，那么超过的高位则不予以处理。

POJ 1676 What time is it?

按位置依次编号为：

   0

   1   2

     3

   4   5

     6

然后先枚举每个位置上可能的数，在枚举每个可能的时间（注意时间最大只能是2359，前两位不能超过23，后两位不能超过59），然后把枚举的时间减去15分钟，再判断是否是第二个时间，即可。

POJ 2015 Permutation Code

模拟题，按题目意思反编码。注意２个: a^b = c 等价于b^c = a。还有这题可以是１个字符，要特殊处理，不然会TLE。

POJ 3699 miniSQL

POJ月赛的一道题，当时其他题目都做不出来，只能整这道模拟了...写了200多行代码，都快吐了...还好最后过了...呵呵。

这题数据有int和string两种，可以都按字符串来输入，比较int时在转成数字。这题的输出比较麻烦，表格的宽度是最长的字符串+2，所以要写扫描符合的信息，求出表格的宽度，再打印。没有什么算法，就是纯模拟，适合用来锻炼代码能力。

POJ 1472 Instant Complexity

一道栈的处理题，难度不大，但要注意几个细节。系数为1的不要输出系数，n的一次方就是n，注意０的情况，还有注意每个样例后面都要换行。
